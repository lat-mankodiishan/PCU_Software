All tasks — priority, period, rationale

Task Prio Period Stack Why this priority + rate
supervisor High (5) 200 ms / 5 Hz 1 KB Highest application prio. Owns the state machine + fault policy + watchdog kick. Pre-empts everything else; must run on time or IWDG resets. 5 Hz matches sensor rate — I_bat fresh data once per loop.
rectifier AboveNormal (4) 5 ms / 200 Hz 1 KB VESC link freshness — sends latest setpoint every tick so VESC sees fresh commands well inside its safety timeout. Highest-frequency task.
pdb AboveNormal (4) 20 ms / 50 Hz 1 KB Contactor GPIO + supervisor heartbeat watchdog. 50 Hz gives 50 ms detection latency on a hung supervisor while still being cheap.
fc_link AboveNormal (4) 50 ms / 20 Hz 2 KB DroneCAN gateway. 20 Hz comfortably handles 5 Hz FlexDebug TX and 1 Hz NodeStatus, plus drains the RX queue with low latency.
can_disp AboveNormal (4) 1 ms poll 2 KB CAN RX dispatch — walks per-bus queues and fans frames to subscribers. Polling at 1 ms is sub-optimal; TODO to switch to ISR thread-flag.
sensor Normal (3) 200 ms / 5 Hz 1.5 KB ADS1262 scan + 3× MAX31855 read. Lower prio than the CAN cluster because data is only consumed by supervisor (also 5 Hz).
ecu Normal (3) 100 ms / 10 Hz 1.5 KB USART2 MegaSquirt 'A' poll. RX timeout 250 ms — blocks via DMA semaphore, so other Normal-prio work isn't starved during the wait.
defaultTask Normal (3) one-shot ~0.5 KB Boot pulse PC3 high 1 s → low → idle forever (osDelay(1000) loop).
dyno_sweep (bench-cfg) bench — Cooperative; supervisor steps aside via expt.active. Not a flight task.
log Low (2) 100 ms / 10 Hz 2 KB SD writes. Lowest acceptable rate that captures all 5 Hz data with margin. f_sync only every 10 writes (1 Hz) to avoid SD flash thrash.
rtos_stats Low (2) 5000 ms / 0.2 Hz 2 KB Periodic CPU + stack HWM dump to SWO. Diagnostic only.
IDLE hook (idle) — — LED toggle every 500 ms. Proves the idle task gets to run (= system isn't 100 % loaded).
