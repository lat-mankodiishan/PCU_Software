pdb_task — current code reference

Role. Bridges g_pt.contactor_cmd to the physical contactor GPIOs, gated by a supervisor heartbeat watchdog. Single watchdog consumer for the supervisor task — no other code raises FAULT_SUPERVISOR_HANG.

Config

DefineValuePurpose
PDB_PERIOD_MS2050 Hz loop
PDB_SUP_HANG_MS10005× margin on 200 ms supervisor refresh; trip threshold

Task priority osPriorityAboveNormal (4), stack 256 words (1 KB).

Internal state

  • last_heartbeat — last seen g_pt.supervisor_heartbeat value
  • last_progress_tick — kernel tick at last heartbeat change
  • supervisor_alive — gate flag; false at boot, latches true on first heartbeat advance

Loop @ 50 Hz

  1. Snapshot supervisor_heartbeat, contactor_cmd.battery, contactor_cmd.rectifier under g_pt_mtx.
  2. If heartbeat changed → update last_progress_tick, set supervisor_alive = true, pt_clear_fault(FAULT_SUPERVISOR_HANG).
  3. Else if supervisor_alive and stale > 1000 ms → supervisor_alive = false, pt_set_fault(FAULT_SUPERVISOR_HANG).
  4. If supervisor_alive: pdb_hw_write(bat_cmd, rect_cmd). Otherwise hold last GPIO state — never open mid-flight.
  5. osDelayUntil(next).

Boot behaviour

GPIO reset state is low = contactors open. supervisor_alive starts false, so the task does not write GPIOs until it has observed at least one heartbeat advance from a running supervisor. defaultTask separately pulses PC3 high for 1 s at boot (precharge latch / startup signal).

Path summary

supervisor (every tick: g_pt.supervisor_heartbeat++ + writes contactor_cmd) → pdb_task (50 Hz: validates heartbeat + mirrors cmd) → pdb_hw_write → physical battery + rectifier contactor GPIOs.
