FC ↔ PCU link — current code reference

Topology. CAN1 @ 1 Mbps, PB8/PB9, TransmitFifoPriority = ENABLE. PCU node 50, FC node 10. PCU's fc_link_task owns the only CanardInstance; no other task touches libcanard. RX path is can_manager dispatch queue → canardHandleRxFrame. TX path is drain_canard_tx() → can_mgr_send(CAN_BUS_DRONECAN, …).

fc_link_task.c (PCU)

Static state

  • s_canard — libcanard instance
  • s_mem_pool[8192] — libcanard heap (8 KB)
  • s_rx_q — 64-deep CAN RX queue
  • Transfer-ID counters for NodeStatus + FlexDebug
  • s_fc_last_status_tick, s_boot_tick

Task loop @ 20 Hz (FC_LINK_PERIOD_MS = 50)

  1. Drain s_rx_q → canardHandleRxFrame (timestamp in µs).
  2. Every 200 ms (5 Hz): publish_flex_debug_batch() — 19 fields.
  3. Every 1000 ms (1 Hz): publish_node_status(uptime).
  4. If s_fc_last_status_tick > 0 and stale > 1000 ms: pt_set_fault(FAULT_FC_STALE).
  5. drain_canard_tx() → can_mgr_send.
  6. canardCleanupStaleTransfers.

scripts/pcu_telem.lua (FC, downlink)

One Lua function update() running every 200 ms (5 Hz). For each of the 19 IDs in the FIELDS table:

  1. ts, buf = DroneCAN_get_FlexDebug(BUS=0, NODE=50, flex_id, last_us[flex_id])
  2. If ts is fresh and #buf >= 4:
    • last_us[flex_id] = ts
    • value = string.unpack("<f", buf) — LE float32
    • gcs:send_named_float(name, value) → MAVLink NAMED_VALUE_FLOAT → MP Tuning view as MAV_<name>, captured in .tlog
    • logger:write("PCU", "Name,Value", "Nf", name, value) → SD-card BIN log row

Banner on script load. No state besides last_us[].

Requires FC params: SCR_ENABLE = 1, CAN_D1_UC_OPTION |= 512 (ENABLE_FLEX_DEBUG) — without this AP's FlexDebug cache stays empty.

scripts/pcu_engine_rc.lua (FC, uplink)

update() every 100 ms (10 Hz):

  1. pwm = rc:get_pwm(RC_CHANNEL = 7)
  2. Classify: < 1300 → OFF (out = 1000), < 1700 → CRANK (out = 1500), else RUN (out = 2000)
  3. SRV_Channels:set_output_pwm_chan_timeout(SERVO_CHAN_0 = 7, out, OVERRIDE_MS = 250) — writes SERVO8 with 250 ms safety timeout (channel reverts if script dies).
  4. AP serialises SERVO8 into uavcan.equipment.actuator.ArrayCommand because SERVO8 is in CAN_D1_UC_SRV_BM. Default command_type = UNITLESS, value = (pulse-1000)/500 - 1 (so 1000 → −1, 1500 → 0, 2000 → +1).
  5. On state change: STATUSTEXT line.

Required FC params: SERVO8_FUNCTION = 94 (Scripting1), SERVO8_MIN = 1000, SERVO8_MAX = 2000, CAN_D1_UC_SRV_BM |= 128, NOT in CAN_D1_UC_ESC_BM (that path is motor-arming-gated).

Path summary (one line per direction)

Downlink (PCU → MP):
g_pt → publish_flex_debug_batch → 5 Hz FlexDebug (DTID 16371) → AP AP_DroneCAN::handle_FlexDebug (gated by ENABLE_FLEX_DEBUG) → cache → pcu_telem.lua polling → gcs:send_named_float → MP Tuning + tlog + dataflash PCU rows.

Uplink — engine state (RC → PCU):
RC7 → pcu_engine_rc.lua → SRV_Channels:set_output_pwm_chan_timeout(SERVO8, …) → AP DroneCAN actuator bridge → ArrayCommand (actuator_id = 8) → fc_link_task.on_transfer_received → pt_request_engine_state() → supervisor (pre-flight gate, then OFF / CRANK direct, RUN via swap+prime).

Uplink — throttle (FC mixer → PCU):
AP motor mixer → ESC RawCommand (all motor channels, motor-arming-gated) → fc_link_task.on_transfer_received → averaged → pt_set_fc_inputs(mode, throttle_pct).

Uplink — param (MP / GUI Tool → PCU):
MP "Configure Parameters" → param.GetSet request → fc_link_task.handle_param_get_set → pt_request_engine_state() (for engine_state) or g_pcu_param_flight_state write (for flight_state).
