PCU firmware — top-down architecture overview

Role. The PCU is the powertrain controller on a series-hybrid hexacopter. A Cube Orange+ Pixhawk runs stock ArduCopter; the PCU (STM32F405) owns: rectifier setpoint to a VESC-based active rectifier, engine throttle servo, ICE engine UART, battery + rectifier contactor sequencing, current + thermocouple sensing, DroneCAN link to the FC, and SD-card logging. State + control law live in the PCU; the FC just hands down throttle demand and engine_state requests.

3. Boot order

  1. HAL_Init
  2. SystemClock_Config
  3. Peripheral MX_*_Init: GPIO, DMA, CAN1, CAN2, DAC, SPI1/2/3, TIM1, FATFS, IWDG, USART2.
  4. __HAL_DBGMCU_FREEZE_CAN1/2 (DEBUG build only — pause CAN on breakpoint).
  5. osKernelInitialize → MX_FREERTOS_Init:
    • pt_init() — zero g_pt, create priority-inherit mutex, park engine throttle at min pulse.
    • can_mgr_init() — TX/RX queues, dispatch task, CAN ISRs armed.
    • Tasks created (order in table below).
  6. osKernelStart — never returns. defaultTask pulses PC3 high for 1 s then idles forever.

7. Engine state machine — supervisor

   ENGINE_OFF      safe_idle()                  duty=0, throttle=0
       │                                        contactors: batt closed, rect closed
       │  pt_request_engine_state(CRANK) via param.GetSet or RC switch
       ▼
   ENGINE_CRANK    crank_hold()                 BLDC, duty=42%, throttle=45%
       │                                        contactors: same
       │  pt_request_engine_state(RUN) OR PA7 rising edge
       ▼
   (swap)          SWAP_BLEED  300 ms           BLDC, duty=0
                   ──(motor swap)──             BLDC→FOC inv
                   SWAP_LOCK   1000 ms          FOC inv, duty=0
                   SWAP_PRIME   500 ms          FOC inv, duty=60%, throttle=60%
       │
       ▼
   ENGINE_RUN      run_fixed_setpoint()         duty=75%, throttle=50%   (current: V1 disabled)
                   run_v1() when RUN_USE_V1_CONTROLLER=1

   FAULT (any time) → SEV_FORCE_FAULT path → ENGINE_FAULT, batt + rect contactors open
   FAULT (any time) → SEV_FORCE_OFF   path → ENGINE_OFF, swap aborted

Pre-flight gate (pt_preflight_ok in powertrain_state.c) rejects CRANK/RUN requests unless VESC + FC alive and no critical faults set. OFF/FAULT always accepted.

9. HAL isolation

Only these files may call HAL_* directly: periph_wrappers.c, can_callbacks.c, can.c/tim.c/spi.c/usart.c/dac.c/iwdg.c/gpio.c/dma.c (CubeMX-generated init), the hardware drivers ads1262.c + max31855.c, and swo_io.c. Application tasks use the wrapper API (esc_hw_set_us, ecu_uart_hw_send/recv, pdb_hw_write, watchdog_refresh, led_hw_toggle, can_hw_is_busoff, etc.).

10. External Lua scripts (run on FC, not PCU)

Two scripts ship in PCU_Code_Ground/scripts/, copied to FC SD card APM/scripts/:

  • pcu_telem.lua — polls AP's FlexDebug cache (DroneCAN_get_FlexDebug, 5 Hz × 19 IDs), forwards to MAVLink NAMED_VALUE_FLOAT + writes PCU rows to dataflash. Requires CAN_D1_UC_OPTION |= 512 (ENABLE_FLEX_DEBUG).
  • pcu_engine_rc.lua — RC7 3-pos switch → SERVO8 PWM (1000/1500/2000) → ActuatorArrayCommand actuator_id=8 → PCU engine state. Requires SERVO8_FUNCTION=94 + CAN_D1_UC_SRV_BM |= 128.

11. Build & dependencies

  • STM32CubeIDE + CubeMX-generated peripheral init. .ioc is the source of truth for CubeMX-managed config.
  • FreeRTOS (CMSIS-RTOS v2 API). Static allocation for tasks (no pvPortMalloc at runtime except libcanard pool).
  • libcanard (vendored under Middlewares/Third_Party/libcanard/).
  • DroneCAN codec headers (vendored under Middlewares/Third_Party/dronecan_codec/). Generated from stock UAVCAN DSDLs; dronecan.protocol.FlexDebug is hand-rolled (encoder lives in fc_link_task.c, decoder via DroneCAN.h).
  • FATFS for SD logging.
  • VESC firmware (sibling repo bldc/): provides the rectifier-side 0x201/0x202 producers and consumes 0x101..0x105.
